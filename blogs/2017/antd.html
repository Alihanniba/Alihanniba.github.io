    # theme-style-css样式拆分

    # 1、全局样式采用css后缀，里面的样式是不经过webpack less module及postcss module 解析的，采用less后缀，页面无法找到相应的class名，且全局样式文件在入口文件导入

    # 2、各模块样式以less后缀单独文件配置，页面中导入样式文件className以对象方式引入，配合postcss module可保证class 唯一性

    # 3、千万不要把全局样式分别写在单个组件中，less 可以使用变量，支持import导入

    通用的样式可用变量表示

    import ‘./index.less’;
