    Node 文件上传踩坑

    花一个无所 4/12/2018 Nodejs前端

    题图：pixabay

    # 需求：

    前端需上传约50M（约50万条数据）csv文件到后端，后端实时逐条解析文件插入数据库。上传时间线下约4-5分钟，线上约1-2分钟。上传过程中http一直 pending。

    # 方案：

    node 中间层，前端文件先上传到node，node 接收文件流，顺便加上请求ticket与其他参数，再往后端传。

    # 坑：

    • node 框架用的koa。koa http 超时时间默认为2分钟，超过2分钟自动abort。
    • 上传文件大小限制

    # Step

    koa-body配置

    import koaBodyParser from 'koa-body';
// 处理文件上传
app.use(koaBodyParser({ multipart: true, formidable: { maxFileSize: 500 * 1024 * 1024 } }));

    如果服务端长时间不反悔数据，存在一个holding 的状态，前端应该做延长超时时间处理。默认的koa http 请求超时时间是2分钟。

    // 设置10分钟超时
	ctx.request.socket.setTimeout(10 * 60 * 1000);

    node 接收前端文件流

    const file = ctx.request.body.files.file;

const formData = {
	//后端参数，attachedName 为文件名，attachedFile为文件流
	attachedName: file.name.split('.')[0],
	attachedFile: fs.createReadStream(file.path),
};

const options = {
	url: url,
	method: 'post',
	headers: {
		'content-type': 'multipart/form-data',
		'sso-ticket': ctx.cookies.get('control-public-ticket'),
	},
	formData,
};
ctx.body = await new Promise((resolve, reject) => {
	request(options, (err, res, body) => {
		if (err) {
			reject(err);
		} else if (body) {
			resolve(body);
		}
	});
});
